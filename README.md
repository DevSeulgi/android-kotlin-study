## Git Commit Message 규칙
| 타입       | 설명                                       |
|-----------|--------------------------------------------|
| **feat**   | 새로운 기능 추가                          |
| **fix**    | 버그 수정                                 |
| **docs**   | 문서 수정 (README.md, 주석 변경 등)       |
| **style**  | 코드 스타일 변경 (포매팅, 세미콜론 추가 등, 기능 변경 없음) |
| **refactor** | 코드 리팩토링 (기능 변화 없이 코드 개선)  |
| **perf**   | 성능 개선                                 |
| **test**   | 테스트 코드 추가/수정                     |
| **chore**  | 빌드, 패키지 매니저 변경 등 (코드 변경 없음) |
| **ci**     | CI/CD 관련 설정 변경                      |
| **revert** | 이전 커밋 되돌리기                        |
